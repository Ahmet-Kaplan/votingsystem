• Al enviar un voto el usuario proporciona un identificador único de voto y un identificador único de solicitud de acceso que son generados calculando el hash de un par de cadenas origen
  • El programa cliente tiene la posibilidad de anular el voto y la solicitud de acceso enviando un documento firmado indicando las cadenas origen.
  • Al anular el voto los servidores hacen pública la solicitud firmada para que los programas que auditan el proceso puedan hacer las validaciones.
  • Una vez anulado el voto el usuario puede volver a votar.
